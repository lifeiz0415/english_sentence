## 앱 구조
- 앱은 `index.html`, `styles.css`, `app.js`, `data.json`으로 구성된 정적 클라이언트 앱입니다.
- 스타일링은 `styles.css` + `index.html`의 Tailwind CDN을 함께 사용합니다.
- 런타임 로직, 상태, 렌더링, 음성, 퀴즈 로직은 모두 `app.js`에 있습니다.
- 시작 시 `bootstrap()`이 `data.json`을 로드하고 `runSelfTests()`를 실행한 뒤 렌더링합니다.

## 데이터 계약
- `data.json` 최상위 키는 `sentences`, `pronunciationMap`, `vocabulary`, `advancedVocabulary`입니다.
- `sentences`는 `English — Korean` 또는 `English - Korean` 형식의 문자열 배열입니다.
- 문장 끝 마침표/물음표는 영어/한글 모두 유지합니다.
- 현재 `DEFAULT_LINES.length`는 365여야 합니다.
- 현재 `vocabulary.length`는 960, `advancedVocabulary.length`는 1958이어야 합니다.
- `vocabulary`/`advancedVocabulary` 항목은 최소 `{ word, partOfSpeech }` 형태이며, `word`는 소문자 영어 단어, `partOfSpeech`는 `noun | verb | adjective | adverb` 중 하나입니다.
- 단어 항목은 선택적으로 `meaning`, `related`, `opposites`를 가질 수 있습니다.
- `parseLines(lines)`는 두 구분자 형식을 모두 지원하고, 각 문장을 `{ id, english, korean, mastered, starred }`로 매핑하며, 한쪽 값이 비어 있으면 제거합니다.

## 정규화 및 채점
- `normalize(text)`는 소문자화 후 영문/숫자/한글/공백만 남기고 문장부호를 제거합니다.
- 문장 타이핑 채점은 **정규화 완전 일치일 때만** 정답 처리됩니다.
- 문장 입력 중 영어 문장은 실시간 문자 단위로 정타(녹색) / 오타(빨간색) 표시를 합니다.
- 입력창은 붙여넣기(`paste`)를 허용하지 않습니다.

## 저장 및 진도
- 빈출문장 저장 키: `const-english-sentences-v4`
- 심화문장 저장 키: `const-english-sentences-advanced-v1`
- 빈출단어 저장 키: `const-english-word-quiz-v1`
- 심화단어 저장 키: `const-english-word-quiz-advanced-v1`
- 문장 진도는 같은 `id` 기준으로 `mastered`, `starred`만 병합합니다.
- 단어 진도는 같은 `word` 기준으로 `mastered`만 병합합니다.
- `resetProgress()`는 현재 문장 모드의 `mastered`, `starred`, 인덱스, 입력 상태를 초기화합니다.
- `resetWordQuizProgress()`는 현재 단어 모드의 `mastered`만 초기화합니다.

## 모드 및 공통 UI
- 활성 모드는 `sentence-basic`, `sentence-advanced`, `word-basic`, `word-advanced` 4개입니다.
- 헤더 메뉴는 `빈출문장 / 심화 문장 / 빈출단어 / 추가 단어` 4개입니다.
- 하단 이동 버튼 문구는 문장 모드에서 `이전문장 / 다음문장`, 단어 모드에서 `이전단어 / 다음단어`입니다.
- 우측 상단 제어 버튼은 `랜덤/순차`, `듣기/조용`, 필요 시 `연속/수동`을 사용하며 on 상태는 연한 녹색, off 상태는 연한 빨간색입니다.
- 우측 상단의 `자동재생` 버튼은 **빈출문장 모드에서만** 보입니다.
- 우측 사이드바 상단에는 `전체 / 암기 / 진도` 카드와 프로그래스바가 표시됩니다.
- `진도` 카드는 현재 모드의 진도 초기화 버튼 역할을 하며, 기본 상태는 `진도/초기화 00%` 한 줄 텍스트, hover 시 `진도 / 초기화` 2줄 텍스트로 바뀝니다.
- 사이드바 높이는 메인 패널의 실제 렌더 높이에 맞춰 동기화되고, 현재 항목은 최상단으로 자동 스크롤됩니다.
- 화면폭이 좁아지면 헤더 메뉴는 햄버거 메뉴로 접히고, 문장리스트/단어리스트 사이드바는 숨겨집니다.
- 화면폭이 좁아지면 `전체 / 암기 / 진도` 카드와 프로그래스바는 헤더 바로 아래로 이동해 표시됩니다.
- 화면폭이 좁아지면 상단의 모드명 + `#번호` 배지는 모드명을 숨기고 `#번호`만 표시합니다.
- 모바일 및 태블릿(화면폭 1024px 미만) 사용자가 접속하면 기본 랜딩 모드는 `빈출단어`입니다.

## 문장 모드 계약
- 앱 첫 로드 시 랜덤으로 선택된 첫 문장을 1회 자동 재생합니다.
- 문장 이동(`이전문장`, `다음문장`, 랜덤, 사이드바 선택) 후 새 문장을 자동 재생합니다.
- 음성은 영어 → 한글 뜻 순으로 읽습니다.
- 암기완료 문장은 학습 대상에서 제외되며, 모두 암기하면 전체 목록을 다시 순환합니다.
- 기본값은 `랜덤 on`, `듣기 on`입니다.

### 빈출문장 모드
- 영어 문장을 그대로 보여주고, 사용자는 전체 문장을 따라 타이핑합니다.
- 영어 문장 표시/숨김 토글을 지원합니다.
- 문장을 숨기면 메인 문장에는 숨김 안내 문구만 보이고, 발음 칩과 사이드바 문장 텍스트는 레이아웃만 유지한 채 보이지 않게 처리합니다.
- 영어 문장 아래에는 단어별 한글 발음 보조 표기를 표시합니다.
- `the` 발음 표기는 다음 단어 첫 글자가 자음이면 `더`, 모음이면 `디`입니다.
- 자동재생 버튼을 사용할 수 있습니다.
- 화면폭이 좁아지면 발음 보조 표기는 숨기고, 한글 뜻 텍스트는 더 작게 표시합니다.

### 심화 문장 모드
- 문장은 항상 보이며, 내용어 중심으로 **항상 2개 단어를 랜덤으로 가려** 보여줍니다.
- 빈칸 후보에서 알파벳 3개 이하 단어는 제외합니다.
- 가능하면 가려진 2개 중 최소 1개는 동사가 되도록 우선 선택합니다.
- 같은 문장을 푸는 동안 빈칸 위치는 유지되고, 다른 문장으로 이동하면 새로 선택될 수 있습니다.
- 가려진 단어는 `s_____` 같은 형태(첫 글자 회색 + 나머지 길이만큼 분리된 `_`)로 표시합니다.
- 현재 문장과 현재 문장 사이드바 항목은 같은 마스킹 규칙을 사용합니다.
- 발음 보조 표기 칩은 표시하지 않습니다.
- 메인 문장 표시창 우측 하단에는 `🔊 다시듣기` 버튼이 있으며, 기본 상태에서는 배경과 자연스럽게 섞여 보이고 hover 시에만 버튼처럼 드러납니다.
- `🔊 다시듣기`는 현재 **영어 문장만** 다시 읽고, 한글 뜻은 읽지 않습니다.
- `다음문장` 버튼을 누르면 가려진 2개 단어의 정답을 메인 문장 영역에서 녹색으로 표시한 뒤, 영어 → 한글 뜻 나레이션이 끝나면 다음 문장으로 이동합니다.
- `듣기 off` 상태에서는 정답 표시 직후 바로 다음 문장으로 이동합니다.

## 단어 퀴즈 모드 계약
- 빈출단어/추가단어 모드는 동일한 레이아웃을 재사용하고, 데이터만 `vocabulary` / `advancedVocabulary`로 나뉩니다.
- 메인 영역에는 영단어 1개와 4지선다 뜻 버튼만 표시하며, 발음 표시는 별도로 노출하지 않습니다.
- 화면폭이 좁아지면 단어 선택지는 2x2 그리드로 표시하고, 상단 단어 표시창 높이와 단어 텍스트 크기를 더 작게 표시합니다.
- `meaning`이 비어 있는 단어는 `뜻 준비중` 상태로 표시됩니다.
- 오답 선택지는 정답 단어와 같은 `partOfSpeech`여야 하며, 가능하면 `related` / `opposites` 기반으로 구성합니다.
- 사이드바에는 단어 목록과 암기 완료 상태를 표시하며, 정답 뜻은 미리 노출하지 않습니다.
- 정답 선택 시 정답 선택지만 녹색으로 표시되고 1초 뒤 다음 단어로 이동합니다.
- 정답 선택 시 단어 나레이션은 다시 나옵니다.
- 오답 선택 시 단어 나레이션은 다시 나오지 않고, 대신 작은 부저음으로 오답을 알립니다.
