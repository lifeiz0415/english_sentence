# AGENTS.md

## 목적
이 문서는 현재 코드베이스(`index.html`, `styles.css`, `app.js`, `data.json`)와 **정확히 일치하는 규칙만** 정의합니다. 구현 지시를 완료할 때마다 이 문서를 실제 코드와 동기화해야 합니다.

## 동기화 규칙 (사용자 요구사항)
- 구현 지시를 완료한 뒤, 동작/데이터 형태/저장 키/UX 흐름이 바뀌었다면 이 파일을 반드시 업데이트합니다.
- 코드에 더 이상 반영되지 않는 규칙은 즉시 삭제합니다.
- 추측성 규칙이나 미래 계획만 있는 규칙은 절대 남기지 않습니다.
- AGENTS.md를 수정한 경우, 같은 작업 사이클에서 누락 없이 즉시 반영하고 변경 사실을 최종 보고에 명시합니다.
- AGENTS.md의 본문/규칙 업데이트는 앞으로 한글로 유지합니다.
- 코드 변경사항이 발생할 때마다 AGENTS.md에 변경사항을 한글 텍스트로 즉시 업데이트하고, 해당 변경을 커밋 대상에 포함합니다.
- 코드 변경사항이 발생할 때마다 해당 변경을 로컬 git에 즉시 커밋합니다.
- 원격 저장소(GitHub) 푸시는 사용자가 명시적으로 요청할 때만 수행합니다.

## 현재 앱 계약 (코드와 반드시 일치해야 함)

### 스택 및 진입점
- 앱은 `index.html` + 같은 레벨의 `styles.css`, `app.js`, `data.json`으로 구성된 정적 클라이언트 앱입니다.
- UI 스타일링은 `styles.css`의 기본 스타일 + `index.html`에서 CDN으로 로드한 Tailwind 유틸리티 클래스를 함께 사용합니다.
- 런타임 로직, 상태 관리, 문장 데이터, 렌더링은 React 런타임 없이 `app.js`에서 구현됩니다.
- 앱 시작 시 `bootstrap()`이 `./data.json`을 fetch하여 문장/발음 데이터를 로드한 뒤 렌더링을 시작합니다.

### 문장 데이터 계약
- `data.json`은 최상위에 `sentences`(문자열 배열), `pronunciationMap`(객체), `vocabulary`(배열), `advancedVocabulary`(배열) 키를 가져야 합니다.
- `sentences`의 각 항목은 `English — Korean`(em dash) 또는 `English - Korean`(hyphen) 형식의 이중언어 문자열이어야 합니다.
- `sentences`의 평서문은 영어/한글 뜻 모두 문장 끝에 마침표(`.`)를 유지하고, 의문문은 물음표(`?`)를 유지해야 합니다.
- `DEFAULT_LINES`는 런타임 시작 시 `data.json.sentences`로 채워집니다.
- `PRONUNCIATION_MAP`은 런타임 시작 시 `data.json.pronunciationMap`으로 채워집니다.
- `WORD_QUIZ_DATA`는 런타임 시작 시 `data.json.vocabulary`로 채워집니다.
- `ADVANCED_WORD_QUIZ_DATA`는 런타임 시작 시 `data.json.advancedVocabulary`로 채워집니다.
- 현재 `vocabulary` 길이는 960개이고, `advancedVocabulary` 길이는 1958개여야 합니다.
- `vocabulary`의 각 항목은 최소 `{ word, partOfSpeech }` 형태여야 하며, `word`는 소문자 영어 단어, `partOfSpeech`는 `noun | verb | adjective | adverb` 중 하나여야 합니다. `meaning`은 나중에 채울 수 있으며, 비어 있는 항목은 현재 단어 퀴즈에서 `뜻 준비중` 상태로 표시됩니다.
- `vocabulary` 항목은 선택적으로 `related`(유의어/근접어 단어 배열), `opposites`(반의어 단어 배열)를 가질 수 있습니다.
- `parseLines(lines)`는 다음을 만족해야 합니다.
  - 두 구분자 형식을 모두 지원
  - 각 항목을 `{ id, english, korean, mastered, starred }` 형태로 매핑
  - 한쪽 값이라도 비어 있는 잘못된 라인은 제거

### 정규화 및 타이핑/채점 동작
- `normalize(text)`는 소문자화하고, 한글 음절/숫자/공백을 제외한 문장부호를 제거합니다.
- 카드 타이핑 채점 로직은 다음을 만족해야 합니다.
  - 빈 입력: 답안을 입력하라는 안내 표시
  - 정규화 완전 일치: 정답 처리 + 암기 완료 표시
  - 유사 정답(부분 일치/임계치): 거의 맞음 피드백
  - 그 외: 오답 피드백

### 로컬 저장 계약
- LocalStorage 키는 `const-english-sentences-v4`를 유지해야 합니다.
- 심화 문장 진도 저장 키는 `const-english-sentences-advanced-v1`를 유지해야 합니다.
- 기초 단어 퀴즈 진도 저장 키는 `const-english-word-quiz-v1`를 유지해야 합니다.
- 심화 단어 퀴즈 진도 저장 키는 `const-english-word-quiz-advanced-v1`를 유지해야 합니다.
- 로드 시:
  - `data.json`에서 최신 기본 문장 본문을 먼저 파싱합니다.
  - 저장 배열이 존재하고 길이가 `DEFAULT_LINES.length` 이상이면 같은 `id` 기준으로 `mastered`, `starred` 진도만 병합합니다.
  - 아니면 기본 문장 파싱 결과로 초기화합니다.
- 문장 상태(`mastered`, `starred`, reset) 변경 시 전체 `sentences` 배열을 저장해야 합니다.
- 단어 퀴즈 로드 시 `vocabulary`/`advancedVocabulary`를 기반으로 퀴즈 아이템을 만들고, 저장된 배열이 현재 길이 이상이면 같은 `word` 기준으로 `mastered` 진도만 병합합니다. `meaning`이 비어 있는 항목은 선택지 대신 `뜻 준비중` 상태로 표시됩니다.

### 내비게이션 및 상태 동작
- `safeSetIndex(next)`는 전체 문장 길이 기준으로 인덱스를 순환(wrap)시키고 `answer`/`feedback`을 초기화합니다.
- 문장 목록이 비어 있으면 인덱스를 0으로 초기화합니다.
- 랜덤 선택은 가능하면 현재 인덱스를 다시 선택하지 않아야 합니다.
- `resetProgress()`는 `mastered`, `starred`를 모두 초기화하고 인덱스/입력/피드백도 함께 초기화해야 합니다.

### UI 동작 계약
- 모드: `sentence-basic`, `sentence-advanced`, `word-basic`, `word-advanced` 4개 활성 모드.
- 헤더 우측에는 `빈출문장 / 심화 문장 / 빈출단어 / 심화 단어` 4개의 메뉴 카드가 표시되며, 현재 사용 중인 메뉴만 강조 상태로 보입니다.
- 앱 최초 로드 시 랜덤으로 선택된 첫 문장을 음성 합성으로 1회 자동 재생합니다.
- 문장 이동(`previous`, `next`, `random`, 사이드바 선택) 후 새 문장을 음성 합성으로 자동 재생합니다.
- 음성 재생은 같은 문장에 대해 영어를 먼저 읽고, 그 다음 한글 뜻을 읽습니다.
- 카드 모드: 한글 뜻 표시/숨김 토글, 영어 문장을 정확히 입력하면 즉시 암기 처리 후 다음 문장으로 자동 이동.
- 심화 문장 모드는 빈출문장 모드와 동일한 레이아웃/동작을 재사용하되, `mastered`/`starred` 진도는 별도 저장 키로 독립 관리합니다.
- 카드 모드에서 영어 문장 표시/숨김 토글을 지원하며, 토글할 때마다 입력 중이던 답안과 피드백을 초기화합니다. 숨김 상태에서는 문장 영역에 숨김 안내 문구만 표시하고, 발음 칩은 레이아웃을 유지한 채 글자만 보이지 않게 처리하며, 타이핑 중 정타/오타 하이라이트를 표시하지 않습니다. 우측 사이드바 예문 리스트는 레이아웃을 유지한 채 글자만 보이지 않게 처리합니다. 빈출문장 모드에서는 이 토글 버튼을 노출하지 않고, 심화 문장 모드에서는 우측 제어 버튼 영역에서 `자동재생` 버튼 왼쪽에 배치합니다.
- 카드 모드 입력 시 영어 문장과 입력값을 실시간 비교해 정타(녹색) / 오타(빨간색) 시각 피드백을 표시합니다.
- 카드 모드 입력창은 기본 상태에서도 검정 테두리(`border-slate-900`)를 유지하며, 포커스해도 테두리 두께/톤이 추가로 진해지지 않습니다.
- 카드 모드 입력창은 붙여넣기(`paste`)를 허용하지 않습니다.
- 카드 모드에서 영어 문장 단어별 한글 발음 보조 표기만 작은 텍스트 칩 형태로 바로 아래에 표시합니다.
- 단어별 발음 보조 표기는 알파벳 철자 읽기(예: 티·에이치·이)가 아니라 실제 발음에 가까운 한글 표기(예: The→더)로 표시합니다.
- 관사 `the`의 발음 보조 표기는 다음 단어 첫 글자가 자음이면 `더`, 모음이면 `디`로 표시합니다.
- 버튼 지원: 영어 문장 표시/숨김 토글, 진도초기화, previous/next, 자동재생 on/off 토글, 랜덤 on/off 토글, 듣기 on/off 토글.
- `전체 / 암기 / 진도` 통계 카드와 프로그래스바는 우측 문장 리스트 상단에 표시됩니다.
- 빈출단어 모드는 기존 메인/사이드바 레이아웃을 재사용하며, 메인에는 영단어 1개와 4지선다 뜻 버튼을 표시하고, 발음 표시는 별도로 노출하지 않습니다. 정답을 선택하면 정답 선택지를 녹색으로 표시한 뒤 1초 후 다음 단어로 이동합니다.
- 빈출단어 모드에서도 `진도초기화` 버튼이 표시되며, 현재 단어 퀴즈 모드의 `mastered` 진도만 초기화합니다.
- 빈출단어 선택지는 가능한 경우 `related` / `opposites`를 우선 사용하고, 부족하면 같은 `partOfSpeech` 안에서 나머지 오답을 채웁니다.
- 빈출단어 퀴즈에서 정답을 제외한 나머지 선택지는 정답 단어와 같은 `partOfSpeech`여야 하며, 가능하면 `related`(유의어/근접어) 또는 `opposites`(반의어) 기반으로 구성합니다.
- 빈출단어 사이드바에는 `vocabulary`에서 로드한 단어 목록과 암기 완료 상태를 표시하며, 정답 뜻은 미리 노출하지 않습니다.
- 심화 단어 모드는 빈출단어 모드와 동일한 레이아웃/동작을 재사용하되, 데이터는 `advancedVocabulary`를 사용하고 진도는 별도 저장 키로 독립 관리합니다. 발음 표시는 별도로 노출하지 않으며, 정답을 선택하면 정답 선택지를 녹색으로 표시한 뒤 1초 후 다음 단어로 이동합니다.
- 심화 단어 모드에서도 `진도초기화` 버튼이 표시되며, 현재 단어 퀴즈 모드의 `mastered` 진도만 초기화합니다.
- 심화 단어 퀴즈에서도 정답을 제외한 나머지 선택지는 정답 단어와 같은 `partOfSpeech`여야 하며, 가능하면 `related`(유의어/근접어) 또는 `opposites`(반의어) 기반으로 구성합니다.
- 암기완료(`mastered=true`) 문장은 학습 대상에서 제외되어 다시 나오지 않습니다. 단, 전체 문장을 모두 암기하면 전체 목록을 다시 순환합니다.
- 앱 시작 기본값: 랜덤 on, 듣기 on.
- 사이드바는 전체 문장 인덱스 선택을 지원하며, 암기완료 문장은 녹색 배경 카드와 `암기완료` 표기로 표시합니다.
- 사이드바 문장 리스트 높이는 전체 레이아웃 높이(`h-full`)를 사용하지 않고, 좌측 메인 카드의 실제 렌더 높이를 측정한 값으로 동기화합니다.
- 표시 문장이 바뀌면(앱 최초 랜덤 문장 표시 포함) 사이드바가 현재 문장을 최상단으로 자동 스크롤합니다.
- 자동재생 동작: 현재 문장을 영어→한글 순으로 읽고 0.1초 대기 후 다음 문장으로 이동합니다. 랜덤 on일 때는 다음 문장을 랜덤으로 선택합니다.
- 자동재생 on 시 듣기는 자동으로 on 상태가 됩니다.
- 듣기 off일 때는 자동/수동 음성 재생을 하지 않습니다.
